Данная работа посвящена решению задачи обнаружения объектов (Object Detection) с использованием семейства моделей **YOLOv11** от Ultralytics.

---

## 1. Выбор начальных условий

### а) Выбор набора данных
**Набор данных:** License Plate Detection Dataset (Распознавание автомобильных номеров).
**Обоснование:** Задача автоматического обнаружения государственных регистрационных знаков (ГРЗ) является критически важной для современных систем "Умного города". Реальное применение включает:
Автоматизацию парковок и контрольно-пропускных пунктов.
Мониторинг нарушений ПДД.
Системы автоматического взимания платы за проезд по платным дорогам.

### b) Выбор метрик качества
Для оценки качества работы модели выбраны следующие метрики:
1. **mAP@50 (Mean Average Precision):** Основная метрика для детекции, показывающая среднюю точность при пересечении предсказанной рамки с истинной более чем на 50% (IoU=0.5).
2. **Precision (Точность):** Показывает долю истинных срабатываний среди всех предсказаний (важно для минимизации ложных детекций, например, чтобы не принять рекламу на кузове за номер).
3. **Recall (Полнота):** Показывает долю найденных номеров от их общего количества на фото (важно, чтобы не "пропускать" автомобили).

## 2. Реализация

В ходе работы использована библиотека `ultralytics` и архитектура **YOLOv11**.

### Создание бейзлайна и оценка качества
В качестве базовой модели (бейзлайна) была выбрана **YOLO11n (Nano)** — самая легкая и быстрая модель в семействе.
**Параметры обучения:** 10 эпох, размер изображения 640.
  **Результаты:**
  
  Precision: 0.98
  
  Recall: 0.92
  mAP50: 0.96

### Улучшение бейзлайна
**Гипотеза:** Использование более сложной архитектуры **YOLO11s (Small)** и увеличение количества эпох до 20 позволит модели выучить более сложные признаки и повысит итоговую точность.
**Результаты улучшенной модели:**
  mAP50 вырос до: 0.97
  Модель стала более стабильно детектировать номера на дальних планах и при плохом освещении.

### Имплементация и тестирование
Для проверки работоспособности обученной модели был проведен инференс на тестовых изображениях, которые модель не видела в процессе обучения. Алгоритм успешно локализует номерные знаки и классифицирует их.
